What is Git?

Git is a Version Control System that helps you:

  • Track code changes
  • Collaborate with teams
  • Restore previous versions
  • Upload code to GitHub

Step 1: Install Git

git --version

Step 2: Configure Git

git config --global user.name "Your Name"
git config --global user.email "your@email.com"
git config --list

Step 3: Initialize Git

git init

Basic Workflow

Working ‚Üí Staging ‚Üí Commit ‚Üí GitHub

Check Status

git status

Add Files

git add index.js
git add .

Commit

git commit -m "Added login feature"

Connect to GitHub

git remote add origin https://github.com/username/repo.git
git branch -M main
git push -u origin main

Daily Commands

1. git pull origin main
2. git checkout -b feature/your-task
3. Work & commit
4. git push origin feature/your-task
5. Create Pull Request
6. Team reviews & merges

  
git pull

üè¢ Working in a Team Project

Step 1: Clone Project

git clone https://github.com/company/project.git
cd project

Step 2: Never Work on Main

git checkout main
git pull origin main

Step 3: Create Your Branch

git checkout -b feature/login-page

Step 4: Work & Commit

git add .
git commit -m "feat: added login page UI"

Step 5: Push Branch

git push origin feature/login-page

Step 6: Create Pull Request

Create a PR on GitHub for review and merge.

Step 7: Keep Branch Updated

git checkout main
git pull origin main
git checkout feature/login-page
git merge main

Daily Team Workflow

1. git pull origin main
2. git checkout -b feature/your-task
3. Work & commit
4. git push origin feature/your-task
5. Create Pull Request

Team Rules

  • Never push directly to main
  • Use branches for each task
  • Pull latest code before working
  • Write clear commit messages
  • Resolve conflicts carefully
  • Test before pushing
üí° Pro Tip: Commit often with meaningful messages like fix: login validation bug
